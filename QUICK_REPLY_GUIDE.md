# LINE Bot Quick Reply 快速選單功能指南

## 功能概述

此更新為 AI Language Bridge LINE Bot 添加了快速選單（Quick Reply）功能，讓用戶能夠在三種翻譯模式之間切換。

### 三種翻譯模式

1. **中日翻譯** (`ja`)
   - 將中文翻譯為日文
   - 默認模式

2. **中英翻譯** (`en`)
   - 將中文翻譯為英文

3. **中英日對照** (`multi`)
   - 同時提供英文和日文翻譯
   - 格式化輸出，清晰區分

## 使用流程

### 步驟 1: 打開快速選單

用戶可以通過以下任一方式觸發菜單：
```
- 輸入: 選單
- 輸入: menu
```

### 步驟 2: 選擇翻譯模式

Bot 會顯示三個快速回覆按鈕：
- 🇯🇵 中日翻譯
- 🌍 中英翻譯  
- 🗣️ 中英日對照

用戶點擊任意按鈕後，該用戶的翻譯模式被設置為所選模式。

### 步驟 3: 發送文本進行翻譯

用戶現在可以發送任何中文文本，Bot 將根據選定的模式進行翻譯。

### 步驟 4: 切換模式（可選）

用戶可以隨時輸入「選單」或「menu」再次打開快速選單，選擇不同的翻譯模式。

## 技術實現細節

### 狀態管理

```python
# 在 app/handler.py 中定義
user_prefs = {}  # {user_id: "ja" | "en" | "multi"}
```

- 每個用戶的翻譯模式獨立存儲
- 默認模式: "ja" (日文)
- 用戶切換模式後，設置立即生效

### 動態 Prompt 系統

根據用戶選擇的模式，系統會生成不同的 System Instruction：

```python
# 日文模式提示詞
"你是一個專業的翻譯助手。請將提供的中文內容翻譯為日文。只返回日文翻譯結果，不需要任何解釋或額外說明。"

# 英文模式提示詞
"你是一個專業的翻譯助手。請將提供的中文內容翻譯為英文。只返回英文翻譯結果，不需要任何解釋或額外說明。"

# 三語模式提示詞
"你是一個專業的多語言翻譯助手。請將提供的中文內容同時翻譯為英文和日文。格式為: 【英文翻譯】 ... 【日文翻譯】 ... 只返回翻譯結果，不需要任何解釋。"
```

## 常見問題

### Q: 用戶關閉 Bot 後，模式設置是否保留？
**A:** 目前模式設置存儲在內存中（`user_prefs` 字典），服務器重啟後會重置。如需持久化，可實現數據庫存儲。

### Q: 多個用戶是否會互相干擾？
**A:** 否。每個用戶有獨立的 `user_id`，模式設置相互隔離。

### Q: 如何添加新的翻譯模式？
**A:** 在 `_get_translation_system_prompt()` 函數中添加新的 elif 分支，並在 `_create_mode_selection_quick_reply()` 中添加對應的按鈕。
